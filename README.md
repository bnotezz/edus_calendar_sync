# 🎓 School Calendar Sync (Edus.school -> Google Calendar)

Автоматизований інструмент для синхронізації шкільного життя з Google Календарем. Більше не потрібно заходити в додаток школи, щоб дізнатися розклад або що сьогодні на обід.

## ✨ Основні можливості

* 📚 **Розумний розклад:** Автоматичний імпорт усіх уроків із зазначенням вчителів.
* 🍽️ **Меню в кишені:** Інформація про сніданки та обіди додається прямо в опис відповідних подій у календарі.
* 🏖️ **Трекер канікул:** Автоматичне створення подій для канікул та державних вихідних.
* 🔐 **Smart Auth:** Автоматичний логін, отримання `UUID` та оновлення токенів доступу. Жодних ручних маніпуляцій у DevTools!
* 🎭 **Візуальні теги (Emoji):** Швидке розпізнавання типу події:
  * 📚 — Навчальні уроки.
  * 🍽️ — Харчування (із меню).
  * 🏖️ — Канікули та вихідні.
* 🤖 **Повна автоматизація:** Працює через GitHub Actions — жодних серверів чи постійно увімкнених ПК.

## 🚀 Швидкий старт

### 1. Підготовка Google Календаря
1. Створіть окремий календар (рекомендується в акаунті дитини).
2. В [Google Cloud Console](https://console.cloud.google.com/) створіть проект, увімкніть **Google Calendar API** та створіть **Service Account**.
3. Завантажте **JSON-ключ** сервісного акаунта.
4. Додайте `client_email` з цього JSON у налаштування доступу вашого Google Календаря з правами "Внесення змін".

### 2. Створення GitHub Personal Access Token (GH_PAT)
Оскільки скрипт автоматично оновлює токени школи, йому потрібен дозвіл на запис у секрети вашого репозиторію. 
1. Перейдіть у **Settings > Developer settings > Personal access tokens > Fine-grained tokens** на GitHub.
2. Натисніть **Generate new token**.
3. У розділі **Repository access** оберіть **Only select repositories** і виберіть цей репозиторій.
4. У розділі **Permissions > Repository permissions** знайдіть **Secrets** і встановіть рівень доступу **Read and write**.
5. Згенеруйте токен і скопіюйте його (він починається з `github_pat_`).

### 3. Налаштування GitHub Secrets
У вашому репозиторії перейдіть до **Settings > Secrets and variables > Actions** та додайте наступні секрети:

| Секрет | Опис |
| --- | --- |
| `GH_PAT` | Токен GitHub, який ви створили на попередньому кроці |
| `SCHOOL_HOST` | Повна адреса вашого порталу (наприклад, `https://vilni-kl.edus.school`) |
| `SCHOOL_LOGIN` | Ваш логін (email) від порталу школи |
| `SCHOOL_PASSWORD` | Ваш пароль від порталу школи |
| `GOOGLE_CALENDAR_ID` | Email календаря (наприклад, `primary` або ID з налаштувань календаря) |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Повний вміст вашого JSON-ключа Google |
| `TELEGRAM_BOT_TOKEN` | (Опціонально) Токен від @BotFather для сповіщень |
| `TELEGRAM_CHAT_ID` | (Опціонально) Ваш ID від @userinfobot |

> **💡 Примітка щодо токенів школи:** Вам **не потрібно** вручну додавати `SCHOOL_TOKEN`, `SCHOOL_REFRESH_TOKEN` або `SCHOOL_USER_UUID`. Система автоматично зробить логін під час першого запуску, витягне ці дані та сама створить відповідні секрети у вашому репозиторії.

## 📅 Графік роботи (Workflows)

Система налаштована на два незалежні цикли:
1. **Щодня о 04:00:** Оновлення розкладу уроків та меню на поточний тиждень.
2. **Щонеділі о 05:00:** Оновлення даних про канікули та семестри.

## 🛠️ Технології

* **Python 3.14+** — основна логіка синхронізації та обробки даних.
* **Google Calendar API v3** — взаємодія з календарем.
* **GitHub Actions & GitHub CLI** — CI/CD для регулярного запуску скриптів та безпечної ротації токенів.
